# Plan: 文件引用指针生成器 (FRP Generator)

> Source PRD: [GitHub Issue #1](https://github.com/nekomorph-woo/cw2_frp/issues/1)

## 架构决策

各阶段共用的架构决策：

- **菜单结构**: 一级菜单「生成 FRP 引用」→ 二级子菜单 #frp1 ~ #frp10
- **触发入口**: 编辑器右键菜单 (EditorPopupMenu) + 项目树右键菜单 (ProjectViewPopupMenu)
- **输出格式**: 5 种格式（文本段、方法、类、文件、文件夹）
- **输出行为**: 自动复制到系统剪贴板 + 显示通知
- **前缀**: 固定使用 `#frp`，不可配置
- **索引范围**: 1-10，用户通过二级菜单选择

---

## Phase 1: 编辑器文本选择（MVP）

**用户故事**:
1. 作为开发者，我想要选中文本后右键生成带行号的引用，以便快速告诉 AI 我要讨论的代码段
5. 作为开发者，我想要通过二级菜单选择索引号，以便在对话中区分多个不同的引用

### 构建内容

实现从文本选择到剪贴板复制的完整端到端路径：

1. 在 `EditorPopupMenu` 注册一级 Action「生成 FRP 引用」
2. 一级 Action 展开二级子菜单，包含 #frp1 ~ #frp10 共 10 个选项
3. 检测编辑器当前选中文本的起始行号和结束行号
4. 获取当前文件相对于项目根目录的路径
5. 拼接生成格式：`#frp{n} to {相对路径} #L{起始行}-{结束行}`
6. 复制到系统剪贴板
7. 显示通知提示（如「已复制: #frp1 to ...」）

### 验收标准

- [x] 在编辑器中选中文本后右键，能看到「生成 FRP 引用」菜单
- [x] 点击一级菜单后展开二级子菜单，显示 #frp1 ~ #frp10
- [x] 选择任意索引后，正确格式的字符串被复制到剪贴板
- [x] 生成的路径是相对于项目根目录的路径
- [x] 行号与实际选中范围一致
- [x] 显示通知提示复制成功

## Phase 2: 编辑器 PSI 上下文识别

**用户故事**:
2. 作为开发者，我想要在方法名上右键生成带方法签名的引用，以便精确指向某个方法
3. 作为开发者，我想要在类名上右键生成文件级引用，以便引用整个类文件

### 构建内容

扩展 Phase 1 的功能，增加 PSI 元素识别能力：

1. 当没有选中文本时，检测光标所在的 PSI 元素类型
2. **方法识别**: 光标在方法声明或方法引用上时，提取方法签名（方法名 + 参数类型列表）
   - 格式：`#frp{n} to {相对路径} #{方法签名}`
   - 示例：`#frp2 to src/main/java/Example.java #resolveValue(String, int)`
3. **类级别识别**: 光标在类/接口/record/enum 声明上时，生成文件级引用
   - 格式：`#frp{n} to {相对路径}`
4. 复用 Phase 1 的菜单结构和剪贴板逻辑

### 验收标准

- [x] 光标在方法名上（无选中文本），生成带方法签名的格式
- [x] 方法签名包含方法名和参数类型（不含返回值）
- [x] 光标在类/接口/record/enum 名上，生成文件级引用格式
- [x] 选中文本优先级高于 PSI 识别（有选中时仍用行号格式）

## Phase 3: 项目树支持

**用户故事**:
4. 作为开发者，我想要在项目树中右键文件/文件夹生成引用，以便快速引用整个文件或目录

### 构建内容

将功能扩展到项目树视图：

1. 在 `ProjectViewPopupMenu` 注册相同的 Action 结构
2. 检测右键目标类型（文件 vs 文件夹）
3. **文件引用**:
   - 格式：`#frp{n} to {相对路径}`
   - 示例：`#frp4 to src/main/java/Example.java`
4. **文件夹引用**:
   - 格式：`#frp{n} to {相对路径}/`
   - 示例：`#frp5 to src/main/java/amp/`
5. 复用 Phase 1 的二级菜单和剪贴板逻辑

### 验收标准

- [x] 在项目树中右键文件，能看到「生成 FRP 引用」菜单
- [x] 选择文件后生成正确格式（无末尾斜杠）
- [x] 选择文件夹后生成正确格式（末尾带斜杠）
- [x] 路径为相对于项目根目录的路径
- [x] 正确复制到剪贴板并显示通知
